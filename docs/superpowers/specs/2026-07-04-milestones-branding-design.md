## `milestones.py` (pure, no I/O)

```
MILESTONES = [(1000,"First Roar"),(5000,"Warming Up"),(10000,"Voice Layer"),
              (25000,"Fluent Flow"),(50000,"Local Legend"),(100000,"Hundred-K Roar"),
              (250000,"Dictation Engine"),(500000,"Thunder Typist"),(1000000,"Million Word Roar")]
```

- `progress(total_words, unlocks=None) -> dict`:
  - `unlocked`: list of `{threshold, name, unlocked_ts}` for every threshold
    `<= total_words`, ascending; `unlocked_ts` from the `unlocks` map (or None).
  - `next`: `{threshold, name}` for the first threshold `> total_words`, or
    None when all are met.
  - `percent`: integer 0–100 progress toward `next` measured from the previous
    threshold (0 if none unlocked yet → from 0); 100 when maxed.
  - `words_remaining`: `next.threshold - total_words`, or 0 when maxed.
  - `total_words`: echoed back.
- `newly_crossed(old_total, new_total) -> [threshold,…]`: thresholds `t` with
  `old_total < t <= new_total`, ascending. Empty when nothing crossed.
- Pure and total: non-int/negative inputs coerce to 0; never raises.

## `history.py`

- Migration to `user_version 3`: `CREATE TABLE IF NOT EXISTS badge_unlocks
  (threshold INTEGER PRIMARY KEY, unlocked_ts REAL NOT NULL)`. The existing v2
  migration path is preserved; v3 block runs after it and bumps the pragma.
- `total_words() -> int`: `SELECT COALESCE(SUM(word_count),0) FROM dictations`
  (ALL-TIME, not the 5000-row insights window).
- `record_unlock(threshold, ts) -> None`: `INSERT OR IGNORE` — first unlock is
  sticky; re-crossing after a history delete never overwrites the timestamp.
- `unlocks() -> {threshold: ts}`.

Progress is computed from the live all-time sum (so deleting history lowers the
current total), but unlocked badges persist in `badge_unlocks` — you keep a
badge you earned even if you later clear history.

## `insights.py`

`compute_insights(rows, now=None, total_words=None, unlocks=None)` gains a
`"milestones"` block = `milestones.progress(total_words if total_words is not
None else words_total, unlocks)`. Backward compatible: existing callers/tests
that omit the new args get a milestones block based on the rows' word sum. The
bridge supplies the all-time total and unlock map.

## `settings_ui.py` bridge

- `get_insights()` passes `total_words=self._history.total_words()` and
  `unlocks=self._history.unlocks()` into `compute_insights`.
- `get_state()` adds `logo_path`.
- `INSTANT_KEYS` += `milestones_enabled`, `milestone_notifications`; both
  bool-coerced in `set_value`.

## `app.py` unlock detection + notification

In `_handle_transcription`, after `record_history` returns a row id AND
`cfg.get("milestones_enabled", True)`:

```
new_total = self.history.total_words()
old_total = new_total - <this dictation's word_count>
for t in milestones.newly_crossed(old_total, new_total):
    self.history.record_unlock(t, time.time())
    if cfg.get("milestone_notifications", True):
        self.notify(f"Milestone unlocked: {name_for(t)} — {t:,} words")
```

Word count for the delta = `len(text.split())` (matches `history.record`).
The whole block is wrapped so any failure logs and is swallowed — dictation is
never affected. Uses the existing tray `notify()` balloon (no new tone, no
overlay). No-op when history disabled (no row id) or milestones disabled.
