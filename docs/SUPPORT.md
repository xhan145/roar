ROAR support is **privacy-safe by default**: we never ask for your transcripts or
audio to help you. Operational details are enough.

## Before you write

- Note your ROAR **version** (Settings → About) and **edition**.
- Reproduce the issue and note what you expected vs. what happened.
- Optionally paste **Safe Diagnostics** (Settings → Diagnostics → Copy). It is an
  allowlist of operational facts only — no transcripts, audio, clipboard, window
  titles, or secrets.

## Bug report template

```
ROAR version:
Edition (Core/Pro/Developer/Supporter):
Windows version:
What I did:
What I expected:
What happened:
Safe Diagnostics (optional):
```

## License issue template

```
ROAR version:
Edition on the license:
Order reference (if any):
Symptom (e.g., "license not recognized"):
verify_license_file.py output (optional; run by support/advanced users):
```

We do **not** need your transcripts, audio, or the contents of your dictation to
resolve support requests. Please do not send them.
