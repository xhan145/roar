# Monetization

ROAR is private, local Windows dictation you own. Monetization funds continued
development **without** compromising the privacy promise.

## Model

- **Buy once, use forever.** No subscription. No account. No cloud transcription.
  No telemetry.
- Purchases include all **v1.x** updates. A future major (v2) may be a separate
  one-time purchase, but versions you bought keep working.
- License validation is **offline** after the license file is imported.

## Editions

| Edition | Price | Positioning |
|---|---|---|
| ROAR Core | Free | Private local dictation |
| ROAR Pro | $19 one-time | Smarter local dictation |
| ROAR Developer Pack | $29 one-time | Code-aware voice layer |
| Supporter License | $49 one-time | Everything in Developer, and supports development |

Paid editions unlock **workflow power** — not basic dignity. See
[FEATURE_MATRIX.md](FEATURE_MATRIX.md).

## Principles (non-negotiable)

- Core dictation is always free.
- **Privacy controls are never paid-only.**
- **Delete-history and delete-audio are never paid-only.**
- Offline use is always free.
- No dark-pattern prompts, no launch nags, no interrupting dictation.
- Licensing code never reads transcript, audio, history, vocabulary, or clipboard
  data.

## First paid beta plan

1. Ship Core free and stable; keep the feature matrix as *policy* (no runtime
   gates) until the licensing path is proven end-to-end.
2. Stand up manual license fulfillment (see [CHECKOUT_SETUP.md](CHECKOUT_SETUP.md))
   for a small paid beta cohort.
3. Collect refund/support signal against [REFUND_POLICY.md](REFUND_POLICY.md).
4. Only then consider enabling any runtime feature gate, with existing users
   grandfathered.
